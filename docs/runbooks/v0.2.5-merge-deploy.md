## Rollback paths

### A. Build verde pero algo se rompe en runtime

Setear `APPROVAL_GATE_ENABLED=0` en Vercel → deshabilita enforcement application-layer. DB sigue protegida por RLS v2 RESTRICTIVE.

### B. RLS v2 niega acceso masivo (usuarios sin claim)

Bajo este escenario hay usuarios con `active_tenant_id` no seteado en JWT. Causa probable: hook custom_access_token no se enganchó o falló al issuance.

Query SQL via Supabase Dashboard SQL editor:

```sql
UPDATE user_session_state
SET active_tenant_id = (
  SELECT tenant_id FROM tenant_users WHERE user_id = user_session_state.user_id LIMIT 1
)
WHERE active_tenant_id IS NULL;
```

Después force-global-signout otra vez → re-login → claim repobla.

### C. Hook custom_access_token crashea token issuance

**Síntoma:** todos los logins fallan con 500 o `Error running hook URI` post Step 8.5 re-enable.

1. Supabase Management API: PATCH `hook_custom_access_token_enabled: false` (~5s)
2. JWT vuelve a no llevar claim → RLS v2 RESTRICTIVE niega, pero v1 PERMISSIVE deja pasar (temporal). Middleware deriva tenant de `user_session_state` table en su lugar.
3. Lord Claudia investiga + fix en branch nuevo + nuevo PR

### E. Migrations W2 fallan al apply en main (Step 8.5)

**Síntoma:** `supabase db push` aborta o aplica parcial.

1. Capturar error exacto + state via `\d+ user_session_state` y `SELECT * FROM supabase_migrations.schema_migrations`
2. Si parcial: investigar qué migration falló + por qué (constraint conflict, dato existente, etc.)
3. Reversal: `supabase db reset --linked` NO usar en prod (destructivo). Mejor: revertir manualmente las migrations que aplicaron OK con DOWN scripts si existen, sino mantener estado y abrir incident response.
4. Hook permanece disabled → magic link sigue funcionando vía SMTP custom Resend, RLS v2 cae a fallback v1 PERMISSIVE.
5. NO reintentar push hasta diagnosticar root cause.

### D. Nuclear rollback

```bash
git checkout main
git revert <merge-commit-hash> -m 1
git push origin main
git tag -d v0.2.5
git push origin :v0.2.5
```

Esto retorna prod a v0.2.4 estado. Vercel auto-deploy. ETA ~5 min.
