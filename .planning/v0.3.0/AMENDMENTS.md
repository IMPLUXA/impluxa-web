## Fix #3 — Cross-shell verify scripts (PowerShell-compatible)

**Rationale:** PLAN.md uses bash heredocs and pipelines (`grep | wc -l | grep -q`) that fail silently in PowerShell. The user's shell is PowerShell per CLAUDE.md env.

### Affected tasks: A2, A3, A5, A6, B5

### Replacement approach

Create `scripts/` directory at repo root with TypeScript verify scripts run via `npx tsx`:

| Old (bash)                                                       | New (cross-shell)                                             |
| ---------------------------------------------------------------- | ------------------------------------------------------------- |
| `grep -rE '(as any\|@ts-ignore)' src/ \| wc -l \| grep -q '^0$'` | `npx tsx scripts/check-no-any.ts`                             |
| `npm run test -- --coverage && node -e "..."` parsing JSON       | `npx tsx scripts/check-coverage.ts --handlers=70 --global=60` |
| Lighthouse + JSON parse                                          | `npx tsx scripts/lighthouse-mobile.ts <url> <host>`           |
| `curl -I \| grep CSP`                                            | `npx tsx scripts/check-security-headers.ts <url>`             |
| `psql ... \| grep`                                               | `npx tsx scripts/verify-pablo-jwt.ts`                         |

### New required scripts (must exist before A3 runs)

- `scripts/check-no-any.ts` — fails if `as any` or unjustified `@ts-ignore` found
- `scripts/check-coverage.ts` — parses `coverage/coverage-summary.json`, asserts thresholds
- `scripts/lighthouse-mobile.ts` — runs Lighthouse mobile profile, asserts category scores
- `scripts/check-security-headers.ts` — curls URL, asserts CSP + HSTS headers present
- `scripts/verify-pablo-jwt.ts` — verifies Pablo's auth state for admin role
- `scripts/check-reviews-resolved.ts` — verifies `docs/reviews/*.md` has no `Status: Open` findings

### Update `vitest.config.ts`

Ensure `coverage.reporter` includes `'json-summary'`:

```ts
test: {
  coverage: {
    reporter: ['text', 'html', 'json-summary'],
    include: ['src/**/*.{ts,tsx}'],
    exclude: ['src/**/*.test.ts', 'src/**/*.spec.ts'],
  }
}
```

This change goes in Wave 1 setup (before A3 starts).
